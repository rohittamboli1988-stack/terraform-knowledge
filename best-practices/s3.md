# S3 Terraform Best Practices

## Bucket Naming
- Globally unique, lowercase, no underscores.
- Pattern: `<org>-<project>-<env>-<purpose>[-<region>]`
- Example: `acme-datalake-prod-raw-us-east-1`

## Security (Non-Negotiable)
- ALWAYS block all public access:
  ```hcl
  resource "aws_s3_bucket_public_access_block" "this" {
    block_public_acls       = true
    ignore_public_acls      = true
    block_public_policy     = true
    restrict_public_buckets = true
  }
  ```
- ALWAYS enable server-side encryption (AES256 minimum, KMS preferred for prod).
- Use bucket policies to restrict access to specific IAM principals only.

## Versioning
- Enable versioning on all non-transient buckets.
- Add lifecycle rules to expire old versions after 30–90 days.

## Lifecycle Rules
- Transition infrequently accessed data to S3-IA after 30 days.
- Archive to Glacier after 90 days if applicable.
- Expire temporary objects (e.g. logs, artifacts) after a defined period.

## Replication
- Enable cross-region replication for DR on critical buckets.

## Access Logging
- Enable server access logging on production buckets.
- Separate log bucket from data bucket.

## Ownership Controls
- Set `object_ownership = "BucketOwnerPreferred"` to avoid ACL confusion.
